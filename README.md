# 🚀 InternSpark Backend Internship

Welcome to my submission repository for the InternSpark Backend Internship. This project focuses on building a production-ready, scalable, and fully tested RESTful API using the Spring Boot ecosystem. 

---

## 📌 Project Roadmap & Tasks

### 1. Build a RESTful API with Spring Boot
* **Goal:** Create a complete CRUD REST API for a resource (e.g., Employee or Product).
* **Requirements:**
  * Use **Spring Web** & **Spring Data JPA**.
  * Implement standard CRUD endpoints with proper HTTP status codes.
  * Use Data Transfer Objects (**DTOs**) and basic request validation.
* **Deliverables:**
  * Clean, documented source code.
  * Postman collection / sample `curl` commands for testing endpoints.

### 2. Database Integration & Migrations
* **Goal:** Connect the application to a relational database and implement version-controlled schema evolution.
* **Requirements:**
  * Design the database schema utilizing Spring Data JPA entities.
  * Apply migrations seamlessly using **Flyway** or **Liquibase**.
  * Seed sample data for testing purposes.
* **Deliverables:**
  * DB scripts and migration files.
  * Configuration details and sample SQL queries.

### 3. Unit Testing & Logging
* **Goal:** Ensure code reliability with a comprehensive test suite and robust application observability.
* **Requirements:**
  * Write unit tests for service classes using **JUnit** and **Mockito**.
  * Configure structured logging via **SLF4J** / **Logback**.
  * Include test reports.
* **Deliverables:**
  * Test suite and coverage reports.
  * Sample log outputs demonstrating crucial system events.

### 4. Dockerize & Deploy
* **Goal:** Containerize the application to ensure consistency across development and production environments.
* **Requirements & Deliverables:**
  * Create a `Dockerfile` to build the application container.
  * Provide deployment documentation or hosted links.

## 📤 How to Submit / Review Tasks

For each completed task, documentation has been prepared following these guidelines:

1. **Task Documentation:** A separate document (DOC, DOCX, or PDF) is created for each task containing code snippets, screenshots, commands, hosted links, and GitHub repository references.
2. **Access:** All files are uploaded to a public cloud storage link (Google Drive / GitHub) with open permissions so anyone can view them.
3. **Submission:** Public links are organized by task number and submitted directly to the official Task Submission page.
